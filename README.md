# Каталог AI-ассистентов

Решение тестового задания на стажировку в Авито.

**Автор:** Чернышков Максим

## Запуск проекта в Docker

```bash
docker compose up -d --build
```

Сервис будет запущен на порту 3000. Backend будет доступен на порту 8080.

## Переменные окружения

Для настройки окружения нужно создать `.env` по аналогии с `.env.example` и поменять значения.

## Тестирование

```bash
make test-backend # unit-тесты backend
make test-integration # интеграционные тесты
make test-frontend # frontend тесты
make coverage # покрытие тестами
```

Покрытие backend составляет 54.3%, frontend 65.6%. 

## Линтер

В качестве линтера на backend используется `golangci-lint`, на frontend `eslint`.

```bash
make lint
```

## Генерация OpenAPI-документации

```bash
make openapi
```

Результат генерации будет в папке `backend/api`.

## Авторизация

`POST /dummyLogin` выдаёт фиксированных пользователей:

- admin: `ca2a62f3-c998-4050-96c1-0c0f62cf6568`, `admin@example.com`
- user: `44c75af3-eca3-4867-85fc-b8245eaafa3a`, `user@example.com`

JWT содержит `user_id` и `role`. Backend проверяет токен и роли централизованно через middleware. При запуске ассистента `user_id` берётся только из JWT.

Поддерживается авторизация по email/паролю, пароли хранятся как bcrypt hash.

## База данных

Используется PostgreSQL. Миграции применяются автоматически при старте backend через `golang-migrate`.

Основные таблицы:

- `users`
- `categories`
- `assistants`
- `assistant_runs`
- `assistant_favorites`

Ключевые индексы покрывают сортировку категорий, фильтрацию ассистентов по категории, активность ассистентов, поиск по названию/описанию, теги, историю запусков по пользователю, фильтры запусков по ассистенту и статусу.

## Запуск ассистента и LLM

Frontend отправляет только пользовательский промпт:

```json
{
  "userPrompt": "курица, рис, томаты, сыр"
}
```

Backend загружает ассистента, проверяет активность, создаёт запуск со статусом `pending`, формирует внутренний LLM-запрос и вызывает mock provider:

```json
{
  "model": "mock-smart",
  "systemPrompt": "Составь простой домашний рецепт из ингредиентов пользователя.",
  "userPrompt": "курица, рис, томаты, сыр"
}
```

Mock provider детерминированно возвращает строку с моделью, пользовательским промптом и системным промптом. Если пользовательский промпт содержит `mock_error`, provider возвращает ошибку. На ошибке provider или пустом ответе запуск сохраняется как `failed`, а API возвращает `502` с `LLM_PROVIDER_ERROR`.

Вызов provider ограничен timeout в 5 секунд.

## Frontend

Реализованы обязательные страницы:

- `/login`
- `/assistants`
- `/assistants/:id`
- `/runs/my`
- `/admin/categories/new`
- `/admin/assistants/new`
- `/admin/assistants/:id/edit`
- `/admin/runs`

Системный промпт обычному пользователю на frontend не показывается: это внутренняя настройка ассистента, которую backend применяет при запуске. Компоненты работают через `frontend/src/services`, URL и заголовки не собираются вручную в страницах.
